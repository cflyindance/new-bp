# 配置中心：一级导航与内部结构融合映射

本文档将《配置中心-完整设计说明》**第五节（配置中心内部结构：大类 → 二级分组）** 融入 **第八节（一级导航 → 配置快捷入口）**，形成业务线侧可落地的**统一信息架构**：快捷入口按「配置中心大类 → 二级分组」归类展示，深链仍指向 canonical 路由。

**关联文档**：`配置中心-完整设计说明.md`（第二、四、五、八、九、十一节）。

---

## 一、使用约定

| 概念 | 说明 |
|------|------|
| **配置中心大类 / 二级分组** | 与第五节表格一致；注册表 `category`、二级域与之一致或可枚举映射。 |
| **快捷入口** | 每条为 **canonical 路由** 上的具体页或 Tab；同一 URL 可在多个业务线下重复出现（双挂策略见主文档第九节）。 |
| **空项** | 某业务线下若某大类无高频项，可不展示该分组，用户从配置中心完整树进入。 |

**合并大类（可选）**：若采用第五节 5.13（如「结账与支付」「订单与客流」），UI 上可将相邻大类**合并为分组标题**，下表行不必删改，仅改变展示层级。

## 三、注册表与 UI 落地（对齐第十一节）

| 字段 | 本方案中的取值 |
|------|----------------|
| `category` | 上表「配置中心大类」对应的第五节枚举/编码 |
| `domain`（或二级域） | 上表「二级分组」，与第五节表格行名一致或可映射 |
| `canonical_route` | 每条快捷入口对应唯一编辑页路径 |
| `quick_entry_modules[]` | 上表「一级导航（业务线）」模块代码，多选 |

**业务线侧渲染建议**：

1. 按 `quick_entry_modules` 命中当前模块的配置项集合；
2. `groupBy(category)` → 组内再 `groupBy(二级分组)` → 列表为深链；
3. 无条目的「大类/二级分组」不展示，避免空分组。

## 四、大类合并展示（对应第五节 5.13）

若配置中心侧栏采用 **6～10 个大类** 的合并版，业务线「相关配置」区可使用相同合并标题，例如：

| 合并后展示名（示例） | 原第五节大类 |
|----------------------|--------------|
| 结账与支付 | 价格、税费与小费；支付与对账 |
| 订单与客流 | 订单与送厨；排队与预约 |

第二节融合表**无需改行**；仅在 UI 分组标题层合并即可。

## 五、交互形态（沿用第八节）

可组合使用：侧栏「相关配置」、模块首页「常用配置」卡片、列表页「⚙ 配置」下拉。结构统一为：**大类 → 二级分组 → 链接列表**。
