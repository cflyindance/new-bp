# 配置中心：完整设计说明

本文档合并原《设置项分类与双入口结构》《功能归类与生效载体》两篇内容，供产品与研发统一对齐：**分类维度**、**双入口**、**功能优先目录**、**生效载体（软件触点 + 硬件角色）**、**功能域与清单聚类**、**注册表字段**与**一级导航快捷入口**。

---

## 一、背景与目标

- **约束 1**：个性化设置项（可达千级）均需纳入**配置中心**，在单一目录体系下可找全。
- **约束 2**：同一设置需在**各一级导航（业务线）**下提供**快捷入口**（深链），缩短操作路径。
- **组织原则**：配置中心目录以**功能配置**为第一维度；**产品线/终端形态**不单独作为侧栏一级分类，仅作为元数据（软件触点）参与筛选与灰显。
- **目标**：客户易懂、实施可配、研发可维护——**单一 canonical 路由** + **多处深链**；同一策略可在多触点、多硬件角色上**同源生效**或按规则覆盖。

## 二、划分维度（行业与客户心智）

### 2.1 为什么不能只按功能名单列平铺

设置项横跨：**设备、资金、票据、流程、人员、门店、菜单、合规、数据**。仅平铺会导致查找慢、培训成本高，且易与「系统设置 / 硬件」等模块重复。

建议采用 **多层维度**：先按「在配置什么」，再按「场景 / 对象」细分。

### 2.2 主维度：按配置对象（配置中心一级分组依据）

| 维度 | 含义 | 典型设置项 |
|------|------|------------|
| **门店与经营场所** | 这家店是谁、在哪、怎么摆场 | 门店信息、外送区域、背景图、平面桌图、座位、等位、预约 |
| **终端与硬件** | 物理设备与通道 | 硬件、终端、收银机、支付设备/终端、打印机、税控机、取餐屏、来电显示、钱箱（偏驱动/绑定侧） |
| **点餐与界面** | 员工/顾客如何点单、如何看屏 | 界面、键盘、搜索、点单规则、订单类型、语言、系统模式 |
| **菜单与商品（配置向）** | 与售卖相关的规则/展示配置 | 菜单设置（与商品中心「编辑商品」区分） |
| **价格与合规** | 怎么计价、合规留痕 | 税、加收、折扣、小费、签名 |
| **支付与对账** | 怎么收、怎么记、怎么对 | 支付方式、支付服务、支付终端（参数侧）、各类支付收据 |
| **打印与票据** | 出什么纸、什么版式 | 打印设置、模板、Runner/Packer/Kitchen/Receipt、等位单、报表单 |
| **订单与厨房流转** | 单怎么生成、怎么送厨、怎么编号 | 订单设置、订单号、送厨、操作设置（流程向） |
| **人员与权限** | 谁能干什么 | 用户、权限、角色与员工 |
| **人力与班次** | 排班、考勤 | 排班与考勤 |
| **系统与安全** | 账号、审计、基础环境 | 系统安全、时间、消息通知 |
| **数据与报表（配置向）** | 汇总口径、默认维度、订阅等 | 销售汇总、商品报表、员工报表、走势、月度经营分析等**配置项**（「看报表」仍在报表中心） |

### 2.3 辅助维度

| 维度 | 用途 |
|------|------|
| **使用角色** | 店长、收银、后厨、实施等——用于**默认权限**、**推荐向导**、帮助文档，不必为每个角色复制一套主导航。 |
| **生效范围** | 集团/品牌默认 vs 单店覆盖；全终端统一 vs 按收银机/站点——建议用**页面内范围选择器**表达，而非在侧栏堆叠。 |

### 2.4 落地规则（避免归类扯皮）

1. **报表 vs 设置**：能改口径/订阅的归「报表配置」；纯查看的归报表中心，配置中心只链过去或只放「默认参数」页。
2. **店 vs 机**：打印机、支付终端、取餐屏优先归**终端/硬件**，并支持按设备配置。
3. **税 / 小费 / 签名 / 加收**：宜聚在**价格与合规**或**支付与结账**，避免散落。
4. **打印**：按**单据类型**分子页（厨房/打包/Runner/收据）比按打印机型号更符合店员语言。
5. **文案统一**：区分「支付终端（通道参数）」与「终端设备（资产/绑定）」，减少客户混淆。

## 三、功能优先原则与生效载体

> 「产品线」仅作为**触点/终端类型**的简称出现在元数据与筛选中，**不单独构成配置中心的一级分类**。不按 POS 一条树、Kiosk 另一条树平行罗列同一业务规则。

### 3.1 组织原则：功能优先，载体为辅

| 层级 | 做什么 | 不做什么 |
|------|--------|----------|
| **配置中心目录** | 按**业务功能**与**二级域**组织，与客户「我要改什么行为」一致 | 不以产品线为主轴复制整棵配置树 |
| **单条 `config_key`** | 定义**行为或策略**（开关、阈值、枚举、绑定关系） | 不为每个终端复制 key（除非行为本质不同） |
| **生效载体（元数据）** | 声明在**哪些触点可读**、**依赖哪些硬件能力**、是否**同源同步** | 不把「仅某端可用」做成与功能平级的独立大章——用载体标签表达 |

### 3.2 同一功能、多载体同时生效（典型建模）

1. **单值多投**：门店/品牌层存一份策略（如外送是否免税）；**POS、自助机、电子菜单、网订**等触点凡涉及计价，**读同一策略键** → 行为一致。  
2. **按硬件角色过滤**：如「自动开钱箱」仅对**绑定了钱箱**的收银类终端生效；未挂钱箱则忽略，无需单独 product key。  
3. **允许按终端覆盖**：同一 `config_key` 定义唯一，**值**可在「门店默认 + 单终端例外」存储（与第二节「生效范围」一致）。

### 3.3 软件触点（用于检索与灰显，非侧栏一级）

| 代码建议 | 含义 |
|----------|------|
| `pos` | 收银 POS |
| `kiosk` | 自助点餐机 |
| `emenu` | 电子菜单（平板等顾客端） |
| `online` | 网上订餐 / 外卖入口 |
| `backoffice` | 商家后台 Web（策略下发、数据、人员） |
| `kitchen_display` | 厨房屏 / KDS（若可独立读策略） |

一条配置可勾选 **多个触点**，表示这些端在具备业务能力时**共用同一配置源**。

### 3.4 硬件角色（能力维度，非品牌型号）

| 代码建议 | 含义 |
|----------|------|
| `receipt_printer` | 收据类打印 |
| `kitchen_printer` | 厨房单打印 |
| `pack_printer` | 打包单打印 |
| `call_display_printer` | 叫号/等位相关打印 |
| `payment_terminal` | 支付终端 |
| `pin_pad` | 外置密码键盘 / PINPad |
| `cash_drawer` | 钱箱 |
| `customer_display` | 客显 |
| `scale` | 磅秤 |
| `caller_id` | 来电显示设备 |
| `dual_screen` | 双屏宿主 |

同一功能项可绑定 **多个硬件角色**（如「支付完成打印」涉及 `payment_terminal` + `receipt_printer`）。

### 3.5 继承链与生效载体的关系（正交）

- **继承链**（集团 / 品牌 / 门店 / 终端）：**值存在哪一层、谁覆盖谁**。  
- **生效载体**：**哪些软件触点读这个值、哪些硬件能力参与执行**。  
二者同时在注册表中维护，互不替代。

## 四、双入口模型（配置中心 + 业务线快捷入口）

### 4.1 原则

| 原则 | 说明 |
|------|------|
| **单一真相** | 每个设置页仅一条 **canonical 路由**（建议 `/config-center/...`；若暂用 `/settings/*` 等需标明迁移计划）。 |
| **快捷入口 = 深链** | 一级导航下只放链接，指向同一路由，不复制页面与表单。 |
| **面包屑** | 从业务线进入仍显示：`配置中心 / 大类 / 具体项`；可提供「本类全部配置」返回配置中心分组。 |
| **配置中心负责「找全」** | 分类 + 搜索/标签；业务线只挂**高频、强相关**项（通常每模块 3～8 个）。 |

### 4.2 路由策略（二选一，需产品定稿）

- **方案 A（推荐长期）**：统一到 `/config-center/*`，旧路径 301/静默重定向。  
- **方案 B（过渡）**：配置中心为**索引 + 外链**，canonical 仍以旧路径为准。

## 五、配置中心内部结构（大类 → 二级分组）

建议 **三级**：**大类（6～12）** → **二级（场景/对象）** → **三级（具体页或 Tab）**。

### 5.1 门店与场地

| 二级分组 | 典型设置项 |
|----------|------------|
| 门店档案 | 门店信息、背景图 |
| 堂食布局 | 平面桌图、座位设置 |
| 外送/配送 | 外送区域 |

### 5.2 终端与硬件

| 二级分组 | 典型设置项 |
|----------|------------|
| 终端总览 | 终端设备设置、收银机、系统模式（若按终端） |
| 支付硬件 | 支付设备、支付终端设置（绑定/驱动侧） |
| 打印硬件 | 打印机设置、税控机设置 |
| 顾客/厨房显示 | 取餐屏设置 |
| 电话 | 来电显示 |
| 现金设备 | 钱箱设置 |

### 5.3 点餐与界面

| 二级分组 | 典型设置项 |
|----------|------------|
| 界面 | 界面设置 |
| 操作效率 | 键盘设置、搜索设置、操作设置 |
| 规则 | 点单规则限制、订单类型 |
| 语言与时间 | 语言设置、时间设置 |

### 5.4 菜单与商品（配置向）

| 二级分组 | 典型设置项 |
|----------|------------|
| 菜单行为 | 菜单设置（与商品中心内容编辑区分） |

### 5.5 价格、税费与小费

| 二级分组 | 典型设置项 |
|----------|------------|
| 税 | 税设置 |
| 加价与优惠 | 加收设置、折扣设置 |
| 结账体验 | 小费设置、签名设置 |

### 5.6 支付与对账

| 二级分组 | 典型设置项 |
|----------|------------|
| 通道与参数 | 支付设置、支付方式、支付服务、支付终端（参数/商户侧） |
| 凭证 | 支付收据（Merchant / CCopy 等） |

### 5.7 订单与送厨

| 二级分组 | 典型设置项 |
|----------|------------|
| 订单主数据 | 订单设置、订单号设置 |
| 厨房协同 | 送厨设置 |
| 流程与操作 | 操作设置（流程向） |

### 5.8 打印与票据

| 二级分组 | 典型设置项 |
|----------|------------|
| 通用 | 打印设置 |
| 模板 | 打印模板 |
| 按单据类型 | Runner、Packer、Kitchen、Receipt、等位单、报表单 |

### 5.9 人员、权限与人力

| 二级分组 | 典型设置项 |
|----------|------------|
| 账号 | 用户设置 |
| 授权 | 权限设置 |
| 组织 | 角色与员工 |
| 班次 | 排班与考勤 |

### 5.10 系统、安全与通知

| 二级分组 | 典型设置项 |
|----------|------------|
| 安全 | 系统安全 |
| 消息 | 消息通知 |

### 5.11 报表与经营（仅配置项）

| 二级分组 | 典型设置项 |
|----------|------------|
| 汇总与口径 | 销售汇总、商品报表、员工报表、走势、月度经营分析等**配置/订阅/默认维度**（可收敛为「报表默认与订阅」一页） |

### 5.12 排队与预约

| 二级分组 | 典型设置项 |
|----------|------------|
| 等位 | 等位设置、等位单（与打印模板可交叉引用） |
| 预约 | 预约设置 |

### 5.13 大类合并建议（可选）

- **价格与合规** + **支付与对账** →「结账与支付」。  
- **订单与送厨** + **排队与预约** →「订单与客流」。  
- 大类数量建议 **6～10** 个，避免侧栏过长。

## 六、功能域细分与清单聚类（含生效载体示例）

以下按**功能**聚类杂乱清单项；**自助机/电子菜单**侧条目并入对应功能域，通过 **第三节** 的触点标签区分呈现端，**不单独开产品线章节**。

每个小节末附 **生效载体示例**：实施时逐 `config_key` 填入注册表。

### 6.1 叫号 / 取餐队列与屏显

| 二级域 | 配置要点（摘录） |
|--------|------------------|
| 队列与单号 | 开关、单号显示时长、数量上限、数据存储时间、付款后移除单号、显示模式 |
| 与网订衔接 | 网上订餐单号区分 |

| 能力 | 典型软件触点 | 典型硬件角色 |
|------|----------------|----------------|
| 队列展示与单号规则 | `pos`、`kiosk`、`online` | 叫号屏、客显（若适用） |
| 单号数据留存 | `backoffice` 下发；各端读展示规则 | 无强制硬件 |

### 6.2 现金、钱箱与营业日

| 二级域 | 配置要点（摘录） |
|--------|------------------|
| 底金与硬币 | 现金底金、硬币卷数量 |
| 平账与误差 | 平账时显示现金销售总额、允许误差、误差提醒 |
| 流程与安全 | 自动登出、操作后登出、营业周期 |
| 钱箱动作 | 刷卡/现金付款时自动开钱箱 |

| 能力 | 典型软件触点 | 典型硬件角色 |
|------|----------------|----------------|
| 现金平账规则 | `pos` 为主；`backoffice` 看报表与策略 | `cash_drawer` |
| 自动开钱箱 | `pos` | `cash_drawer`、`payment_terminal`（按通道） |

### 6.3 订单生命周期、桌台与流程

| 二级域 | 配置要点（摘录） |
|--------|------------------|
| 开台与人数 | 跳过选桌/人数、每单最多客人、座位显示、换桌、开单前换桌 |
| 送厨与付款时机 | 点餐是否直接送厨、点击送厨/付款送厨、结账/打单后送厨 |
| 订单编辑与支付 | 编辑应收、部分支付编辑/取消、小数数量、减菜重定向 |
| 分单与展示 | 未分单菜品展示、相同菜拆分/合并、菜序 |
| 搜索与辅助 | 搜索菜单、自定义点单、自动点完套餐、已送厨改调味 |
| 单状态与原因 | 失效/删单原因、删单是否通知厨房、未付款删除、按菜退款 |
| 模式与时间 | 盘点、餐厅/菜单模式、清桌、点单超时、ASAP、24 小时制 |
| 网订/会员展示 | 加入会员、显示手机号 |

**顾客自助侧**（订单类型页、取餐方式、牌号、手机/姓名必填、隐私条款、一桌多单限制、用餐/下单次数、火锅规则、呼叫服务员、下单前授权、海报/视频/抽奖等）均归入本子能力，用 **`kiosk` / `emenu` / `online`** 区分呈现。

| 能力 | 典型软件触点 | 典型硬件角色 |
|------|----------------|----------------|
| 税/折扣/送厨等业务规则 | `pos`、`kiosk`、`emenu`、`online`（点单触点） | 通常无 |
| 桌台/企台流程 | 以 `pos` 为主；`emenu` 企台只读等 | 无或 `customer_display` |
| 火锅/多轮下单限制 | 多为 `kiosk`、`emenu` | 无 |
| 呼叫服务员 | `emenu` 为主 | 可选打印机/屏 |

### 6.4 税、加收、折扣、小费、服务费

| 二级域 | 配置要点（摘录） |
|--------|------------------|
| 税 | 加收是否计税、折扣后计税、外送免税、总价四舍五入 |
| 加收与促销 | 合单加收重算、整单/餐具/打包加收、子单促销重算 |
| 套餐与调味 | 套餐子菜价格展示、默认全局调味价 |
| 小费 | 算法、建议比例、分享小费、隐藏现金小费、多次支付按整单小费、默认小费值 |
| 卡与签名门槛 | 信用卡最低金额、签名最低金额 |
| 服务费 | 税后服务费、清除线上订单服务费 |
| 折扣合规 | 折扣原因必填、自定义原因 |

| 能力 | 典型软件触点 | 典型硬件角色 |
|------|----------------|----------------|
| 计税与四舍五入 | `pos`、`kiosk`、`emenu`、`online` | 无 |
| 小费/签名流程 | 各触点收银闭环 | `pin_pad`、`payment_terminal`、`receipt_printer` |
| 加收/折扣原因 | 各触点下单/改价 | 无 |

### 6.5 支付体验与收据

| 二级域 | 配置要点（摘录） |
|--------|------------------|
| 流程 | 刷卡签名、小费与签字流程 |
| 收据 | 纸质/电子/短信、Customer/Merchant Copy、批量结算打印报告、支付图标 |
| PINPad | PINPad 上加小费/签名 |
| 部分付款 | 部分付款自动打印订单收据 |

| 能力 | 典型软件触点 | 典型硬件角色 |
|------|----------------|----------------|
| 收据策略 | `pos` 为主；`kiosk` 同源可共用 | `receipt_printer`、`payment_terminal` |
| PINPad | `pos` | `pin_pad` |

### 6.6 界面与操作布局（含「更多」矩阵）

| 二级域 | 配置要点（摘录） |
|--------|------------------|
| 主操作可见性 | 删单/送厨/付款/分单/小费等隐藏到「更多」 |
| 网订历史 | 历史单界面隐藏复制等 |
| 自定义 | 分割线名称 |
| 组/类/菜/调味 | 平铺、类展示、菜展示、调味展示 |
| 通用界面 | 默认主界面、默认键盘、字体与主题（自助/电子菜单） |

建议用**一页矩阵**管理大量「隐藏到更多」。

| 能力 | 典型软件触点 | 典型硬件角色 |
|------|----------------|----------------|
| POS 主界面按钮矩阵 | `pos` | 无 |
| 自助/电子菜单展示 | `kiosk`、`emenu`（可同 key、分触点默认） | 无 |

### 6.7 打印策略与设备绑定

| 二级域 | 配置要点（摘录） |
|--------|------------------|
| 打印机属性 | 类型、IP、多语言名、切纸、备用机 |
| 按用途绑定 | 收据/支付/叫号/上菜/打包/等位/报表/自定义菜/机器人通知等 |
| 支付终端 | 品牌、型号、IP、端口、通讯方式、关单打印收据、运行模式 |
| 其他外设 | 来电、磅秤、客显、双屏、叫号屏、设备管理器端口 |
| 模板与语言 | 菜单多语言、通用打印设置 |

| 能力 | 典型软件触点 | 典型硬件角色 |
|------|----------------|----------------|
| 全局打印路由 | `backoffice` 下发；`pos`/`kiosk` 执行 | 各 `*_printer` |
| 单终端覆盖 | 值落在**终端级** | 具体设备实例 |

### 6.8 门店主数据、人员、数据与集成

| 二级域 | 配置要点（摘录） |
|--------|------------------|
| 门店与形象 | 餐馆信息、营业时段、平面图、区域、LOGO/封面/打印 Logo |
| 组织与人力 | 员工、角色、出勤、班次、工资、销售统计入口 |
| 记录与比例 | 付款记录、钱箱进出场、小费支出、折扣/加收比率 |
| 数据留存 | 与队列、报表留存统一为「数据管理」 |
| 集成 | LevelUp、7shifts、Ingenico、WorldPay 等 |

| 能力 | 典型软件触点 | 典型硬件角色 |
|------|----------------|----------------|
| 门店主数据 | `backoffice` 维护；全端读 | 无 |
| 集成 | `backoffice` | 视集成绑定 `payment_terminal` 等 |

### 6.9 工具栏 / 快捷键分组（POS）

分单、删单、保存、付款、送厨、备注、税、调味、改价、排序、换桌、整单备注、会员、打单、加收、折扣、小费、整单税等——**布局配置**单独成页。

**生效载体**：主要为 `pos`；其他触点若有工具条可复用 schema、`layout_profile` 区分。

## 九、交叉项与双挂策略

- **钱箱、订单类型、折扣**等可能跨类：配置中心 **只归一个主类**；业务线允许多个快捷入口指向 **同一 canonical URL**。  
- **背景图**：门店形象与点餐屏可共页 Tab 分场景，或两入口同 URL。

## 十、配置中心 UI 与检索（功能优先）

1. **侧栏**：展开到**功能大类 → 二级域**（几十级以内）。  
2. **列表页筛选**（次要）：**软件触点**、**硬件角色**、门店、终端、关键词——不替代功能分类。  
3. **详情页**：上部功能与表单；下部或侧栏 **「生效范围与载体」**（触点、继承链、硬件角色）。  
4. **场景包**：快餐/正餐/火锅等预设，一键写多 key。

## 十一、配置项注册表与落地交付物

### 11.1 注册表字段（建议）

| 字段 | 用途 |
|------|------|
| `config_key` | 全局唯一、稳定键 |
| `title` / `title_i18n`、`description` | 展示名与帮助文案 |
| `category` | 配置中心大类（对应第五节） |
| `domain` | 功能域/二级域编码（对应第六节） |
| `canonical_route` | 唯一编辑页路径 |
| `value_type` | bool / int / enum / json / device_ref 等及校验规则 |
| `software_touchpoints[]` | `pos` / `kiosk` / `emenu` / `online` / `backoffice` / `kitchen_display` |
| `hardware_roles[]` | 第三节硬件角色（可为空） |
| `sync_policy` | 如 `unified`、`touchpoint_default`、`terminal_override_allowed` |
| `inheritance` | 集团 → 品牌 → 门店 → 终端 |
| `search_tags[]` | 对象（门店/终端）、场景（打印/支付）等，便于检索 |
| `quick_entry_modules[]` | 一级导航快捷入口（多选） |
| `conflicts_with[]` / `requires[]` | 互斥与依赖 |

原 `applicable_surfaces` 建议并入 **`software_touchpoints`**；硬件用 **`hardware_roles`** 表达。

### 11.2 权限表

每项配置对应 `read/write` 与角色，避免「配置中心能进、业务线快捷进不去」不一致。

---

## 十二、清单治理建议

1. **去重**：同一语义合并为一个 `config_key`，用 `sync_policy` + 继承处理差异，不按终端复制 key。  
2. **模糊项**：业务确认后再挂载体（如「清泉」等）。  
3. **多处「名称」**：绑定实体类型（打印机实例、终端、门店），避免 key 冲突。

---

## 十三、总结

- **配置中心**：以 **对象 + 场景（功能）** 树形收纳全部设置，配 **搜索/标签**；目录**不以产品线为主轴**。  
- **生效载体**：用 **软件触点 + 硬件角色** 说明「同一配置在哪些端、哪些设备上生效」；与 **继承链** 正交。  
- **一级导航**：每模块 **相关配置短列表** 均为 **深链**，与配置中心 **同路由**。  
- **交叉项**：**主类唯一**，**入口可多**。
