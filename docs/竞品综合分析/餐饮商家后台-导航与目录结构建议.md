# 餐饮行业商家后台管理平台 — 导航与目录结构建议

> 面向美国市场，结合美国竞品（Toast、Chowbus、Square）、国内餐饮 SaaS（美团、饿了么、企迈、天财商龙）及国内连锁标杆（麦当劳、肯德基、蜜雪冰城）的竞品分析与导航设计建议。

---

## 一、竞品分析摘要

### 1. 美国市场

#### Toast

Toast 以「单一平台驱动整店」为理念，按**产品套件**划分导航，而非单点功能：

| 套件 | 包含方向 |
|------|----------|
| **POS & Restaurant Operations** | 收银、堂食/外卖/电话/在线订单统一处理 |
| **Digital Storefront Suite** | 官网订餐、品牌页、SEO、订单追踪、促销 |
| **Marketing Suite** | 客人触达、复购、忠诚度、营销活动 |
| **Restaurant Management Suite** | 多门店、运营配置、报表与决策 |
| **Supplier & Accounting Suite** | 供应商、成本、会计对接 |
| **Payroll Suite** | 薪资、工时、小费、报税 |
| **Team Management Suite** | 排班、小费分配、入职与权限 |
| **Restaurant Grade Hardware** | 硬件设备与配置 |
| **Finance** | 贷款、即时到账等金融产品 |

特点：**按业务域（Operations / Marketing / Team / Finance）分层**，每个套件下再拆子功能，适合多门店与连锁。

> **深度拆解**：Toast **Toast Web** 后台左侧导航与配置树，已根据官方 Platform Guide 与《Access permissions reference》中的路径逆向整理，见 **[Toast-Web-后台结构深度分析.md](./Toast-Web-后台结构深度分析.md)**（含 Menus / Kitchen / Front of house / Takeout & delivery / Payments / Reports / Employees / Toast account 等完整子树与 IA 特征）。

#### Chowbus

Chowbus 商家端分为 **Chowbus Go（商户 App）** 与 **POS Dashboard** 两套入口，模块划分如下：

- **订单**：堂食/外卖/官网开关、在线与配送订单统一处理  
- **菜单**：菜品上下架、多渠道同步  
- **员工**：绩效、权限  
- **数据**：经营概览、时段/订单类型/菜品/员工维度报表、日结  
- **多门店**：门店切换、数据汇总  
- **KDS**：厨显、前后场协同  
- **支持**：工单、帮助中心  

特点：**以「订单 + 菜单 + 人 + 数据」为主线**，轻量、偏单店与中小型。

#### Square for Restaurants

- 主导航以 **Items & Menus** 为核心，下面再分 Menu Groups、多渠道（POS / 在线 / Kiosk / 配送）。  
- 强调**同一菜单多场景**（店内、线上、自助）统一配置与报表。  

特点：**以「商品/菜单」为中心**，再延伸订单、报表、设置。

### 2. 国内市场

#### 美团（餐饮）

- **收银 / 点餐**：收银系统、自助点餐、扫码点餐  
- **订单**：外卖接单、订单提醒、统一订单列表  
- **会员与营销**：会员、储值、积分、发券宝、新客/生日等营销  
- **供应链**：进销存  
- **数据**：经营报表、客流、诊断  
- **评价**：平台评价与回复  
- **设置**：门店、外卖、通用设置；云喇叭、推广码、WiFi 等  

特点：**外卖与到店并重**，强营销与评价，偏平台生态。

#### 饿了么商家版

- **订单**：接单、改状态、退单、筛选  
- **菜单/商品**：上下架、价格、描述、库存  
- **店铺**：名称、地址、营业时间、推广  
- **营销**：满减、折扣券、限时抢购、会员  
- **数据**：销售、订单量、热销/滞销、活动效果  
- **客服**：在线沟通、评价回复、投诉  

特点：**以「订单 → 商品 → 店铺 → 营销 → 数据」** 为动线，偏外卖单一场景。

#### 天财商龙 / 企迈（餐饮 SaaS）

- **POS 与客位**：收银、支付、验券、挂单、桌台  
- **点餐**：小程序、手持 APP、扫码点餐、团购核销  
- **会员与营销（SCRM）**：储值、积分、权益卡、RFM 营销、优惠券  
- **后厨 KDS**：排单、并单、急推、出品顺序  
- **库存（SCM）**：出入库、条码、保质期  
- **财务中台**：业务与财务协同、对接用友/金蝶  
- **连锁总部**：多店报表、账户、监管、供应链、央厨  

特点：**到店 + 连锁 + 供应链 + 财务** 完整闭环，导航偏「收银 → 点餐 → 会员 → 厨房 → 库存 → 财务」。

### 3. 国内连锁标杆：麦当劳、肯德基、蜜雪冰城

以下为国内快餐/茶饮连锁实际使用的系统与后台能力，侧重**门店/店长/加盟商后台**与**总部—门店—供应链**一体化，对设计「连锁型商家后台」导航有直接参考价值。

#### 麦当劳中国

- **规模**：近 7000 家门店、约 20 万员工，日均订单数百万级，会员超 3 亿。
- **点餐与前台**：自研 APP + 小程序 All-in-one（十余个小程序合并），到店取餐「百秒级」；自助点餐机、扫码点餐、柜台点餐并存；取餐号 + 大屏叫号（客显屏/取餐屏）。
- **POS 与运营**：每笔订单驱动数百条供应链与库存逻辑，数据实时同步至全国门店前台与生产/配送；侯餐时间控制在约 210 秒内。
- **供应链**：与菜鸟合作 RFID，包材「生产—物流—餐厅」全程数字身份证；餐厅日盘存由约 1 小时降至约 15 分钟，库存准确率提升约 30%；整体供应链效率提升约 40%+。
- **门店后台（店长/值班经理）**：  
  - **排班与考勤**：按历史销售、员工表现、天气等智能推荐班表；刷脸/指纹/APP 打卡；请假、调班。  
  - **订货与库存**：与 POS 联动，支持基于营业额的人手与物料预估。  
  - **绩效与培训**：培训课程、绩效评估、学习路径。  
  - **沟通协作**：即时通讯（如 WorkJam）、任务指派、问题反馈、文件共享。  
  - **人事**：员工档案、薪资福利、招聘到离职全流程（HRIS）。
- **数据**：门店级差异化经营，按区位（学校、写字楼、机场等）精准推送活动。

**对导航的启示**：**排班/考勤/绩效/培训/沟通**与**订货/库存/报表**并重；「取餐叫号/客显系统管理」作为运营标配；供应链与门店数据强打通，适合在「运营」或「报表」下预留「订货/库存/盘点」入口。

#### 肯德基 / 百胜中国（含必胜客等）

- **系统定位**：定制化 **ERP**，采购、库存、销售、财务、人力、供应链一体化，全国 1.6 万+ 门店数据实时可控。
- **核心模块**：  
  - **采购**：与供应商对接、自动下单、物料跟踪。  
  - **库存**：门店与仓储实时监控、智能补货预警；原料临期自动预警并补货。  
  - **销售与订单**：门店点单、中台结算、销售分析。  
  - **财务**：自动账目与多维度财务监控。  
  - **人力**：排班、考勤、绩效、薪酬。  
  - **供应链协同**：从原材料到出库全链路追踪。
- **店长权限**：人员（出勤、服务、培训、奖惩）、**订货与缺货**、**损耗**（内外部）、收银与报表、卫生/促销/安全。
- **成效**：库存周转率提升约 20%+，单店日盘点由约 40 分钟降至约 10 分钟；全国促销调价约半天完成。办公侧引入 WPS 365 等降本。

**对导航的启示**：**订货/缺货/损耗**是店长端高频能力，应在一级或二级导航中显性存在（如「运营 → 订货与库存」或「报表 → 损耗」）；**收银与报表**合并为店长工作台核心入口。

#### 蜜雪冰城

- **规模**：全球约 4.6 万家门店（以加盟为主），2024 年前三季营收约 187 亿元；供应链覆盖内地约 90% 县城 12 小时达、约 97% 门店冷链。
- **收银与点餐**：哗啦啦（部分门店）+ 自研；扫码、小程序预购、柜台多入口；单店日均千单级仍可稳定运行。
- **供应链**：自研系统，补货从约 3 天缩至约 6 小时；按销量预测自动生成采购计划；5 大生产基地、核心食材 100% 自产。
- **加盟商/门店后台（蜜雪通等）**：  
  - **订货**：按近 7 天销量、天气、节假日生成「建议订货量」，加盟商确认即可；手机 + PC 报货、线上支付；配送路线优化、12 小时配送圈。  
  - **经营数据**：各店销售、库存实时；手机/PC/收银端多端报表。  
  - **库存与原料**：精确到克的原料配比、售卖自动扣减库存。  
  - **巡店与监控**：AI 摄像头（卫生、操作规范）、违规推送整改、移动巡检与汇报。  
  - **出餐**：KDS 分配订单、显示原料用量与操作时长、超时预警。
- **数据分析**：观远数据等打通 ERP、POS、CRM、外卖，支持会员/营运/贸易分析，统计效率显著提升。
- **协同**：飞书覆盖总部、加盟商、门店、供应商。

**对导航的启示**：**订货/补货建议**（智能订货）作为独立子模块更符合连锁与加盟场景；**巡店/监控/巡检**可归入「运营」或「多门店管理」；**加盟商视图**（只看本店/本区域）与总部视图（全部门店）需在 Dashboard 与权限上区分；**损耗/库存**与报表、订货放在同一逻辑块内便于店长操作。

### 4. 连锁标杆与通用 SaaS 的导航差异小结

| 维度 | 麦当劳/肯德基/蜜雪（连锁自营或强管控） | 美团/饿了么/天财（通用 SaaS） |
|------|----------------------------------------|-------------------------------|
| **订货/补货** | 店长/加盟商核心能力，智能建议订货、缺货预警 | 多放在「供应链」或「库存」，非首屏 |
| **排班/考勤** | 与销售预估、人手配置强绑定，独立模块 | 多为「团队」下子项 |
| **损耗** | 店长必看（内外部损耗、盘点差异） | 报表中有，少单独入口 |
| **巡店/巡检** | 总部—区域—门店，AI 监控 + 移动巡检 | 少见或为独立产品 |
| **加盟商视图** | 仅本店/本区域数据与订货 | 多店切换多为「同品牌多店」 |
| **供应链可见性** | 从原料到出库、配送时效、盘点时长 | 进销存为主，深度因产品而异 |
| **取餐/叫号/客显系统管理** | 标配（麦当劳取餐号+大屏） | 常见于「设备」或「运营」 |

若产品面向**连锁/加盟**（含美国连锁品牌），建议在导航中显性保留：**订货与库存**（含智能补货建议）、**损耗报表**、**巡店/巡检**（可选）、**多角色视图**（总部/区域/店长/加盟商）。

## 二、美国市场与国内差异（对导航的影响）

| 维度 | 美国（Toast/Chowbus） | 国内（美团/饿了么/天财商龙） |
|------|------------------------|------------------------------|
| 用工与合规 | 排班、小费、Payroll、报税独立成块 | 多为「员工/权限」一块，少有独立 Payroll 套件 |
| 营销重心 | 品牌官网、邮件/SMS、忠诚度计划 | 平台券、满减、直播、评价运营 |
| 订单来源 | 官网 + 第三方聚合 + 堂食 + 电话 | 平台外卖 + 到店 + 小程序并重 |
| 多门店 | 多店切换、汇总报表、权限分店 | 连锁总部、门店监管、供应链分店 |
| 金融 | 贷款、即时到账、Toast Checking 等 | 多为平台结算、对账，少独立金融产品 |

因此，**面向美国市场的导航**需要：

- 把 **Team / Payroll / Tips** 提到一级或二级重要位置。  
- **Digital Storefront / Online Ordering** 独立成块（官网 + 多渠道订单）。  
- **Marketing** 按「品牌 + 忠诚度 + 促销」组织，而非仅「发券」。  
- 保留 **Operations（订单/菜单/厨房/多店）** 与 **Reporting / Finance** 的清晰分区。

## 三、推荐的一级导航（目录）结构

结合竞品与美国习惯，建议**一级导航 7–9 个**（符合 Hick’s Law），子级不超过 2 层。

### 方案 A：按「业务域」划分（偏 Toast，适合中大型、多门店）

```
1. Dashboard（首页/工作台）
   - 今日概览、待办、快捷入口、关键指标

2. Orders（订单）
   - All orders / Dine-in / Online / Delivery / Pickup
   - Order history / Refunds & voids

3. Menu & Items（菜单与商品）
   - Menus / Menu groups / Items & modifiers
   - Availability by channel / By location

4. Operations（运营）
   - Kitchen display (KDS) / Table management
   - Multi-location switch / Location settings

5. Guests & Marketing（客人与营销）
   - Loyalty / Rewards / Gift cards
   - Promotions / Email & SMS / Digital storefront (online ordering)

6. Team & Payroll（团队与薪资）
   - Team / Roles & permissions / Scheduling
   - Payroll / Timesheets / Tips & tip-out

7. Reporting & Finance（报表与财务）
   - Sales reports / Labor / Product mix / Locations
   - Settlements / Payouts / Finance (capital, instant deposit)

8. Settings（设置）
   - Business profile / Locations / Tax / Integrations
   - Hardware / Notifications / Support
```

### 方案 B：按「任务流」划分（偏 Chowbus/饿了么，适合单店或轻连锁）

```
1. Dashboard

2. Orders
   - Live orders / History / Refunds

3. Menu
   - Items / Categories / Availability / Sync channels

4. Kitchen & Floor
   - KDS / Tables / Service

5. Marketing & Loyalty
   - Promotions / Loyalty / Online ordering page

6. People
   - Staff / Scheduling / Payroll / Tips

7. Reports & Money
   - Sales & labor reports / Settlements / Finance

8. Settings
```

### 方案 C：混合（突出「数字化门店」与「人」）

在方案 A 基础上，把 **Digital Storefront / Online Ordering** 单独成一级（若产品线较重），或作为 **Guests & Marketing** 下的首要子模块并配明显入口。

## 四、二级目录（子导航）建议

在确定一级后，二级按「对象 + 操作」或「场景」组织，避免超过 2 层深度。

| 一级 | 二级示例 | 说明 |
|------|----------|------|
| **Orders** | List / Filters / Refunds / History | 按状态/类型筛选，退单单独入口 |
| **Menu & Items** | Menus / Items / Modifiers / Availability / By location | 与 Square 类似，强调「按渠道/按门店」 |
| **Operations** | KDS / Tables / Locations / Printers / Terminals | 设备与多店放在一起 |
| **Guests & Marketing** | Loyalty / Promotions / Gift cards / Online ordering / Email-SMS | 官网订餐与营销并列 |
| **Team & Payroll** | Team list / Roles / Scheduling / Timesheets / Payroll / Tips | 小费与 Payroll 同级 |
| **Reporting & Finance** | Sales / Labor / Product / By location / Settlements / Payouts / Capital | 经营报表与资金分开 |
| **Settings** | Business / Locations / Tax / Integrations / Hardware / Notifications | 支持多门店与第三方 |

### 4.1 报表中心：二级与三级目录草案（美团 × Toast 融合 · 中英对照）

> **设计原则**：二级按**经营任务**划分（而非照搬某一家产品命名），同时兼容 **美团式「平台漏斗」**（流量→转化→履约→评价→营销）与 **Toast 式「门店 P&L」**（销售多维拆解→菜单结构→人工→支付与小费→多店）。  
> **与本文 §八 的对应**：下列三级已尽量覆盖脑图「收入/菜品/损耗/餐段/钱箱/礼品卡/月结/异常/打款/会员/优惠券」及独立模块「支付服务」「资金周转」的报表入口；**团队管理 → 员工报表** 仍以人力明细为主，本处「人工与排班」侧重**汇总类经营报表**（可与 Team 模块互链）。

#### 4.1.1 二级导航总表（建议 8～10 项，按业态显隐）

| 二级导航（中文） | Secondary (EN) | 典型用户 | 显隐建议 |
|------------------|----------------|----------|----------|
| 经营概览 | Overview & KPIs | 老板、店长 | 默认展示 |
| 销售与渠道 | Sales & channels | 老板、店长、运营 | 默认展示 |
| 流量与转化 | Traffic & conversion | 运营、外卖负责人 | **有平台外卖/官网投流时展示** |
| 商品与菜单 | Items & menu mix | 老板、厨师长、运营 | 默认展示 |
| 履约与服务 | Fulfillment & service | 店长、配送负责人 | **有外卖/自营配送时展示**；纯堂食可改为「出餐与服务时效」或弱化 |
| 人工与排班 | Labor & scheduling | 老板、店长 | 美国市场默认展示；可与 **团队管理 → 员工报表** 互链 |
| 成本与库存 | Cost & inventory | 老板、店长 | **有进销存/订货/损耗时展示** |
| 营销与会员 | Marketing & loyalty | 运营、市场 | 默认展示（券/会员/礼品卡效果） |
| 支付与结算 | Payments & settlements | 老板、财务 | 默认展示 |
| 评价与口碑 | Reviews & reputation | 店长、运营 | **接平台评价或第三方评价聚合时展示** |

```
8. 团队管理 (Team)（`admin-web`：二级在侧栏「团队管理」下折叠展开，交互同智能点餐）
   - 角色与员工 (Roles & employees)
   - 休息与加班 (Breaks & overtime)
   - 员工打卡 (Clock-in)
   - 小费管理 (Tip management)（主区内三级：小费分配、分配明细、分配规则；访问 `/team/tips` 默认落到小费分配）
   - 员工报表 (Employee reports)（主区内三级：概览、小费、绩效、薪资；路由 `/team/reports/...`，访问 `/team/reports` 默认落到概览；交互布局同小费管理）
   - 7Shifts 对接（排班）
   - 排班与考勤（连锁）（主区内三级：考勤记录、加班规则；路由 `/team/scheduling/...`，访问 `/team/scheduling` 默认落到考勤记录；交互布局同小费管理）

8.1 **权限管理** (Access control)（`admin-web` 一级 `/permissions`，侧栏二级，**交互同团队管理**；**侧栏顺序**：紧邻「设置」之上，便于与「账号与权限」等设置项相邻）
   - **权限总览**：当前角色/员工赋权概况
   - **角色与功能权限**：配置各 **角色** 可访问的后台模块与操作（查看、编辑、审核等）
   - **员工授权**：指定 **哪位员工** 拥有哪些角色，从而能管理或操作后台哪些能力
   - **权限变更记录**：赋权与策略调整留痕；与「设置 → 账号与权限」可分工（本模块侧重 RBAC 矩阵与员工绑定）

9. 报表与财务 (Reports & Finance)
   - 收入报表 / 收入明细 / 餐段分析
   - 菜品分析 / 损耗报表
   - 钱箱报表 / 交易异常报表
   - 礼品卡报表 / 会员报表 / 优惠券报表
   - 月结单 / 打款与费用报表
   - 资金周转：独立一级或并入报表（Capital / 即时到账等）
   - **二级与三级导航（中英对照、美团 × Toast 融合、与脑图映射）**：见 **§4.1 报表中心**。

9.1 **支付服务** (Payment services)（`admin-web` 一级 `/payment-services`，侧栏二级：支付概览、支付渠道、结算与到账、风控与争议）

9.2 **消息通知** (Notifications)（`admin-web` 一级 `/notifications`，侧栏二级：消息中心、通知设置、模板与订阅）

9.3 **打印模板** (Print templates)（`admin-web` 一级 `/print-templates`，侧栏二级：模板列表、模板设计；一级点击展开/收起、再选子项，交互同智能点餐）

9.4 **设备管理** (Device management)（`admin-web` 一级 `/device-management`，侧栏二级：设备总览、**硬件**、终端管理、绑定与授权、监控告警；其中 **硬件** 在主内容区为左侧细项导航：支付、钱箱、路由器、POS、POS GO、KDS、叫号屏、打印机、电子秤、Kiosk、eMenu，路由前缀 `/device-management/hardware/...`，交互同 Kiosk 点餐左侧导航；原一级「设备」`/operations/devices` 已移除，书签重定向至本模块）

10. 设置 (Settings)
    - 二级交互：与 **设备管理** 一致，一级「设置」点击展开/收起后，在左侧切换二级
    - **设置总揽**（`admin-web` `/settings/overview`）：展示支付、钱箱、路由器、POS、POS GO、KDS、叫号屏、打印机、电子秤、Kiosk、eMenu 等快捷入口，跳转至 **设备管理 → 硬件**（`/device-management/hardware/...`）
    - 基础设置（门店信息、营业时间、多门店）
    - 报表设置 / 打印与票据 / 小费设置 / 服务流程
    - 安全设置 / 账号与权限 / 操作日志
    - 集成与 API（含外卖平台、7Shifts 等）
```

若需**严格 7–9 个一级**，可合并为：

- 将 **交易** 并入 **报表与财务**（报表与财务 → 下设「交易流水」）。
- 将 **设备** 并入 **设置**（设置 → 设备与硬件）。
- 则一级为：**主页 / 订单 / 菜单 / 智能点餐 / 运营 / 促销与会员 / 团队管理 / 报表与财务 / 设置**（共 9 项）。

#### 8.1 连锁/加盟场景补充（对标麦当劳、肯德基、蜜雪）

面向**连锁或加盟**业态时，在现有 9 个一级下可增加或强化以下二级，与国内麦当劳、肯德基、蜜雪的能力对齐：

| 一级 | 建议新增/强化二级 | 对应标杆能力 |
|------|-------------------|--------------|
| **运营** | **订货与库存**（智能补货建议、缺货预警、报货/配送） | 肯德基 ERP 订货、蜜雪「蜜雪通」建议订货量、麦当劳库存联动 |
| **运营** | **巡店与巡检**（移动巡店、AI 监控、违规整改） | 蜜雪 AI 摄像头 + 巡店、肯德基店长卫生/安全 |
| **报表与财务** | **损耗报表**（内外部损耗、盘点差异、钱箱） | 肯德基店长损耗管理、麦当劳盘点 15 分钟、蜜雪库存扣减 |
| **团队管理** | **排班与考勤**（按销售/天气的智能排班、打卡、调班） | 麦当劳智能排班与 WorkJam、肯德基人力模块 |
| **设置** | **角色与数据范围**（总部/区域/店长/加盟商视图、仅看本店/本区域） | 蜜雪加盟商只看本店、百胜全国 1.6 万店统一视图 |

## 十、国内连锁标杆能力与导航映射

将麦当劳、肯德基、蜜雪冰城的能力拆解为「后台应有模块」，并映射到推荐导航，便于 PRD 或排期时查漏。

| 标杆能力 | 推荐归属 | 说明 |
|----------|----------|------|
| 智能排班（销售/天气/人手预估） | 团队管理 → 排班与考勤 | 麦当劳、肯德基店长核心 |
| 订货/补货建议、缺货预警、报货 | 运营 → 订货与库存 | 肯德基 ERP、蜜雪蜜雪通 |
| 损耗管理（内外部、盘点差异） | 报表与财务 → 损耗报表 | 肯德基店长手册、麦当劳盘点 |
| 巡店、AI 监控、违规整改、移动巡检 | 运营 → 巡店与巡检 | 蜜雪标配 |
