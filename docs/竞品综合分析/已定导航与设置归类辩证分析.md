## 1. 分析方法论（辩证框架）

| 层次 | 问题 |
|------|------|
| **正题** | 当前索引表对每条设置项给出的「结论（一级导航）」是否与已定 27 个中心一致、且子类命名自洽 |
| **反题** | 竞品 IA 的共性约束：一级不宜过多、SSOT、打印≠硬件、小费/税/营业时间单一主入口、渠道矩阵 |
| **合题** | 在**不推翻已定导航**前提下：哪些归类**已合理**、哪些需**更名映射**、哪些需**补归档**、哪些需在**设置中心 UX** 层做聚合/双链 |

## 2. 总体结论（Executive Summary）

### 2.1 量化覆盖

| 指标 | 结果 |
|------|------|
| 索引已归档设置项 | **587** 条 |
| 可归入已定 27 导航 | **575** 条（98.0%） |
| 索引使用但不在 27 导航中的名称 | **12** 条（**平台业务中心** 5 + **素材管理中心** 7） |
| 已定导航但索引为 0 条 | **5** 项：主页、评价中心、信贷中心、素材中心、配置中心 |

### 2.2 总体评价

| 维度 | 评分 | 说明 |
|------|------|------|
| **与竞品域模型对齐度** | ★★★★☆ | 支付/财务/打印/硬件/前厅/后厨/外卖/会员/促销拆分符合 Toast/Square/Peblla 最佳实践 |
| **索引与已定导航一致性** | ★★★☆☆ | 存在 2 处命名漂移（平台业务中心、素材管理中心）；5 个已定中心尚无设置归档 |
| **SSOT / 防双源** | ★★★☆☆ | 小费、薪资报表、品牌 LOGO、菜单模式等已在索引「为何归入」中标注风险，但产品层尚未全部收口 |
| **导航粒度** | ★★★☆☆ | 27 个一级偏多（竞品建议 12～14），适合多产品线 B2B；需靠**配置中心/主页**做聚合，避免商户迷路 |

**合题判断**：已定导航的**领域切分方向正确**，与索引归档**大体合理**；当前主要问题不是「归错中心」，而是 **(1) 命名与已定导航未完全对齐**、**(2) 部分中心空置**、**(3) 订单中心体量过大**、**(4) 门店/配置/素材的 SSOT 入口需在产品上写清**。

## 4. 逐项辩证：27 个已定一级导航

### 4.1 经营与组织层

#### 品牌管理（索引 8 条）— **合理，宜加强与子模块边界**

| 辩证 | 内容 |
|------|------|
| 正题 | 品牌层级、品牌-菜单映射、服务商 LOGO 露出、对外联系信息归品牌，符合 Square「品牌推广」与 Toast 企业信息分层 |
| 反题 | 竞品将 LOGO **位图**放素材库、**规范**放品牌；索引已把餐厅 LOGO 放素材中心（7 条待更名），品牌侧保留「模式/映射/露出策略」 |
| 合题 | ✓ 保持品牌管理；设置中心内分：**品牌架构** vs **视觉素材（链素材中心）**；避免在品牌页直接上传小票热敏稿 |

#### 门店管理（索引 4 条）— **方向对，覆盖偏薄**

| 辩证 | 内容 |
|------|------|
| 正题 | 营业时段、国家/地区、正餐快餐模式、营业结束提醒 — 属门店 SSOT |
| 反题 | 竞品（Clover/Snackpass）将**营业时间、时区、对外展示**均放在「店铺/门店」；索引中大量「门店相关」散落在：外卖/来取（线上营业）、系统设置（营业周期）、支付/税务 locale |
| 合题 | ⚠ **归类原则合理，归档不完整**；建议在索引后续批次将「门店档案、时区、对外电话地址（若与品牌不同）」收口到门店管理，外卖/来取仅保留**渠道特殊营业时段 override**（竞品 P2：全局默认 + 渠道覆盖） |

#### 主页（索引 0 条）— **合理**

| 辩证 | 内容 |
|------|------|
| 合题 | ✓ 主页作为**经营看板/快捷入口**，不承担设置 SSOT；索引中 Dashboard 相关（默认主界面等）归系统设置或品牌「品牌作为首页」是**壳层路由**问题，可不强行并入主页 |

#### 团队管理（16）vs 权限管理中心（15）— **合理**

| 辩证 | 内容 |
|------|------|
| 正题 | 考勤、排班、薪酬与小费分配、强制登出条件（部分）分属团队 vs 权限，符合 Square 职员/权限拆分 |
| 反题 | 索引中「经理强制登出」「企台须付清才能登出」归权限；「小费池分配政策」应在团队；**支付侧小费算法**在支付中心 — 三角关系清晰 |
| 合题 | ✓ 保持双中心；设置中心用交叉链接：团队·小费分配 ↔ 支付·小费流程 ↔ 报表·小费计算标准 |

## 7. 设置中心 UX 建议（不改一级导航）

每个一级导航下的「设置中心」建议统一三层（对齐竞品 Snackpass + Peblla）：

```
设置中心
├── 全渠道默认（SSOT）
├── 渠道覆盖（线上 / 店内 POS·Kiosk·CDS / 第三方）
└── 高级 / 集成（链到配置中心）
```

**订单中心、前厅管理中心**设置项多，必须用 **子 Tab + 影响范围徽章**（全渠道 / 仅 POS / 仅 Kiosk），否则 27 个一级仍显冗长。

**配置中心**建议作为商户侧的「**集成与发布**」聚合页，承接索引中平台类 5 条 + 未来发布下发，减轻在 POS 高级设置里找集成的问题。
