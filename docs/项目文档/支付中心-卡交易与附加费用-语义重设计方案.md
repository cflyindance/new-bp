# 支付中心 · 卡交易与附加费用 — 语义重设计方案

> 文档版本：v1.1（已执行：242 产线门槛、454 互斥加价；243→客显；307→财务；512/543 排除）  
> 依据：配置项**标题 + 场景描述**语义分析（不依赖旧分组文档）  
> 范围：当前 `card-fees` 组 6 条（242 / 243 / 307 / 454 / 512 / 543）；已排除 **452**  
> 目标：组名与内容一致、消除重复 SSOT、对客加价策略互斥、迁出非「卡费」语义项

---

| 动作 | seq | 说明 |

## 2. 现状语义诊断

### 2.1 四条语义带（当前混在一组）

```text
A. 能否用卡付（门槛）     → 242、512  （重复）
B. 签名合规（交互）       → 243       （非「费用」）
C. 对客加价（顾客多付）   → 454、543  （应互斥，文案撞车）
D. 商户成本（对内口径）   → 307       （非对客定价）
```

### 2.2 组名问题

「卡交易与**附加费用**」只准确描述 **C**；**A** 是支付方式约束，**B** 是结账合规，**D** 是财务/报表口径。

## 4. 分项方案

### 4.1 242 ∪ 512 — 合并为单一门槛 SSOT

| 项 | 内容 |
|----|------|
| **保留 seq** | **242** |
| **新标题** | 信用卡最低消费金额 |
| **场景描述** | 订单金额低于该值时，不可选择信用卡支付（全产线统一，与支付方式矩阵中「信用卡」行联动说明）。 |
| **排除** | **512**（与 242 同义，原散落「服务设置」） |
| **UI** | 金额数字输入 + 单位说明；可选在 234 支付方式矩阵「信用卡」行脚标只读展示该阈值 |

**数据迁移**：若本地已有 `512-*` storage，读取时 fallback 合并到 242 字段一次。

---

### 4.2 243 — 签名金额门槛（迁出「卡费」）

| 项 | 内容 |
|----|------|
| **归属** | 支付中心 **`cds-checkout-ux`**（SSOT 宿主组） |
| **组内位置** | 签名区块**最前**：**243 → 8 → 464**（门槛 → 行为 → 客显显隐） |
| **新标题** | 信用卡签名最低金额 |
| **场景描述** | 卡交易金额达到该值时要求电子签名；**全终端生效**（客显 / 扫码 / Paypad 签名开关见本组及下方两组）。 |
| **理由** | 配置的是**合规交互**，不是费率；与 8/464/497 同问题域 |

**UI**：金额输入；组内 243 行下增加一句灰色说明：「低于该金额且终端已开启签名页时，可跳过签名。」

**不必**在三个结账组各复制一条；243 在客显组展示即可，扫码/Paypad 组 intro 交叉引用。

---

### 4.3 307 — 收单成本率（迁出支付 hub）

| 项 | 内容 |
|----|------|
| **归属** | **财务中心** · `fees-tips-expense`（费用折扣与小费支出） |
| **组内位置** | 组末，与 305/451 等「费用项」并列 |
| **标题保持** | 信用卡交易手续费比例(%) |
| **场景描述（重写）** | 用于报表与成本核算的**收单通道费率**（商户付给收单方/通道）；**不改变**顾客结账应付金额。对客加价见支付中心「卡付加价策略」。 |
| **理由** | 原导航在「报表」；与 454/543 **对客加价** 混配会导致商户配反 |

**hub override**：在 `settings-hub-override.mjs` 将 307 从支付中心改挂财务中心（若终版 B 平台导航为报表，设置滑层 SSOT 仍建议 **财务**，因与 305/451 费用核算同链路）。

---

### 4.4 454 + 543 — 卡付加价策略（互斥 SSOT）

#### 4.4.1 产品语义

| 模式 | 原 seq | 顾客感知 | 监管备注（美国等） |
|------|--------|----------|-------------------|
| **无加价** | — | 现金与卡同价 | 默认 |
| **双重定价** | 454 | 标价/现金价 vs 卡价 | Cash discount / dual pricing 披露 |
| **整单加收** | 543 | 卡付时在总额上加 % | Credit card surcharge 披露与上限 |

**454 与 543 不可同时启用**；452 已排除（与 454 重复的占位项）。

#### 4.4.2 Catalog 策略

| seq | catalog 展示 |
|-----|--------------|
| **454** | **唯一滑层行**，标题改为 **卡付加价策略** |
| **543** | **排除 catalog**（逻辑并入 454 UI；storage 可保留 `543-*` 作 legacy 或统一写入 454 JSON） |

#### 4.4.3 专用 UI 草图

**展示 seq**：454（单行占满策略面板）

```text
┌─ 卡付加价策略 ─────────────────────────────────────────┐
│  ○ 不加价（现金与卡付同价）                              │
│  ○ 双重定价 — 卡付价 = 现金价 × (1 + ___) %             │
│       说明：菜单/小票展示现金价与卡价（合规披露文案另配）   │
│  ○ 整单加收 — 选卡付时在订单总额上加 ___ %               │
│       说明：与订单中心「加收」预设（447）不同，仅卡付触发  │
│                                                          │
│  [比例输入框]  仅在选中后二选一之一时启用，0–4% 或辖区上限  │
└──────────────────────────────────────────────────────────┘
```

**存储建议**（localStorage / 后端字段）：

```json
{
  "mode": "none" | "dual-pricing" | "surcharge",
  "percent": 3.5
}
```

- `fieldId`: `454-card-pricing-strategy`（JSON）
- 读取 legacy：`454` 旧值 + `543` 开关/比例 → 迁移为上述结构（543 开 → `surcharge`，454 有比例 → `dual-pricing`）

**与订单 447 边界**（组首或 454 面板内一句）：

> **447** 为服务员手动/规则触发的**通用加收**；**本项**仅在支付方式为信用卡时自动套用，二者可并存但小票行项需区分展示。

### 10.2 商户配置路径（心智）

```text
支付中心
  支付方式          → 能用什么付、哪些终端
  卡支付与加价      → 能不能刷卡、刷卡要不要加价
  结账与交互       → 签名门槛(243) + 签名页 + 流程
财务中心
  费用与小费支出    → 通道成本率(307)，算报表用
订单中心
  加收              → 通用加收预设(447)，与卡付自动加收无关
```
