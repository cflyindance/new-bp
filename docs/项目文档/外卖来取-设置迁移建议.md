# 外卖/来取设置迁移建议（仅基于配置语义）

## 目标

让“外卖/来取”只承载履约与交付体验相关配置；把技术接入、堂食流程、打印支付合规配置迁回对应中心。

- 建议保留：5 项
- 建议迁移：5 项
- 建议合并：3 组

## 迁移判定规则

配置若主要回答“如何配送/自取并完成交付”，放外卖/来取；若主要回答“系统怎么连通、堂食怎么服务、凭证怎么打印”，迁往对应专业中心。

## 一、建议保留在外卖/来取

| Seq | 功能 | 建议归属 | 判断依据 |
| --- | --- | --- | --- |
| 429 | 外送区域设置 | 外卖/来取 | 配送履约能力本体（服务范围） |
| 546 | 按菜品收取打包盒费用 | 外卖/来取 | 外卖成本与计费策略 |
| 489 | 送餐/取餐服务方式 | 外卖/来取 | 履约模式配置（自取/送餐） |
| 490 | 展示取餐方式 | 外卖/来取 | 食客流程开关，属于履约体验 |
| 491 | 打包展示输入号码牌 | 外卖/来取 | 自取交付闭环（核销识别） |

## 二、建议迁出外卖/来取

| Seq | 功能 | 建议迁入 | 原因 |
| --- | --- | --- | --- |
| 95-106 | 线上服务 host/port/callback/hash | 系统设置/集成中心 | 技术接入参数，非门店运营语义 |
| 93 | 网上点餐免开端口设置 | 系统设置/集成中心 | 网络部署能力，偏运维 |
| 98 | self dine-in host URL | 前厅管理中心 | 语义为堂食能力，不是外卖/来取核心 |
| 94 | 收据打印确认签名栏 | 打印中心或支付中心 | 支付凭证与打印合规能力 |
| 503 | 送餐到桌餐牌号获取方式 | 前厅管理中心 | 服务员送餐到桌偏堂食前厅流程 |

## 三、建议合并与去重

| 涉及项 | 当前问题 | 合并建议 |
| --- | --- | --- |
| 31 + 487 | 订单类型 / 选择订单类型 | 合并为一个“订单类型可用范围”设置，避免双入口同义 |
| 488 + 490 | 展示订单类型选择页 / 展示取餐方式 | 改为层级开关：先“是否展示类型页”，再“是否展示取餐方式页” |
| 489 + 503 | 服务方式 / 餐牌号获取方式 | 保留依赖关系：仅当“送餐到桌”启用时展示 503 |

## 四、落地优先级

- P0：先迁出 `95-106、93`（技术参数），避免业务页承载运维配置。
- P1：再迁出 `98、503`（堂食前厅语义），理顺域边界。
- P2：最后迁出 `94` 并完成 `31/487、488/490` 去重，减少认知负担。
