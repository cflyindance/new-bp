# 打印中心 · 设置二级导航重设计方案

> 文档版本：v1.10（「收据版式与明细」拆为「收据明细与价格」8 条 +「收据版式与辅助」6 条）  
> 数据范围：打印中心设置 catalog **34** 条可见  
> 竞品参考：Toast / Clover / Square / Peblla / Snackpass 商家后台结构文档

---

## 1. 背景与目标

### 1.1 现状问题

| 指标 | 现状 | 问题 |
|------|------|------|
| 二级分组 | **8 组** / 34 条 | 原单组 14 条版式项拆为「明细与价格」+「版式与辅助」 |
| 商户心智 | 触发 / 执行 / 版式 混排 | 配置「何时打」与「印什么行」易混淆 |

### 1.2 设计目标

- v1.10：版式类 **14 条** 拆为 **收据明细与价格（8）** + **收据版式与辅助（6）**
- v1.9：**265** 迁入 **打印基础**；**收据打印执行** 仅 **262/273**
- 新增 **收据打印执行**（份数、快速模式、仅新菜）
- **打印基础** 吸收打印前选机（269/270）
- 跨 hub：小费/未付价/税位 → 支付中心；线上签名栏 → 前厅；堂食标识 → 打包组
- **不修改** `配置归类-终版.md` 原文

## 2. 竞品对照（打印/小票设置维度）

| 竞品 | 打印设置组织方式 | 本项目借鉴 |
|------|------------------|------------|
| **Toast** | 打印机与钱箱、账单小票设置、打印机/单据/KDS | 支付收据与厨房单分轨 |
| **Clover** | 订单小票、支付小票、打印偏好、设备打印机 | 收据流程、执行、版式三层 |
| **Square** | 打印机配置文件、收据打印选项、订单票与标签 | 触发逻辑与版式字段独立 |

### 2.1 打印设置七维（商户心智）

```text
打印基础 → 订单触发 → 支付收据 → 打印执行 → 明细与价格 → 版式与辅助 → 打包单 → 单号小票

**边界**：厨房单 → 后厨；平台备注/外送份数 → 外卖/来取；小费/未付价/税位 → 支付中心；C 端签名栏 → 前厅。
```

## 4. 分类结果明细

### 4.1 打印基础（`print-foundation-devices`）

**归类理由**：页高/Logo/轮打/265 快速打印、269 选机（POS/PayPad/POS Go）；全局打印引擎与设备路由，非票面字段。

| seq | 场景 | 原导航 | 功能模块 | 功能设置 | 功能场景描述（摘要） |
|-----|------|--------|----------|----------|----------------------|
| 167 | 收据 | 其他 | 基本设置 | 打印页高 | 设置单张小票打印最长大小 |
| 256 | 收据 | 打印 | 基本设置 | 打印Logo | 设置哪种类型的收据上是否打印LOGO |
| 259 | 收据 | 打印 | 基本设置 | 启用轮打 | 设置当请求打印服务失败时，是否一直轮询请求打印机进行打印 |
| 265 | 收据 | 收据打印 | 打印设置 | 快速打印收据模式 | 是否开启快速打印收据模式 |
| 269 | 收据 | 收据打印 | 打印设置 | 桌面POS打印前选择打印机 | 设置桌面POS打印前是否需要选择打印机 |

### 4.2 订单收据触发（`order-receipt-trigger`）

**归类理由**：654 下单后、500 部分付款后，按产线配置是否在门店收据机自动打订单收据；与支付收据流程分轨。

| seq | 场景 | 原导航 | 功能模块 | 功能设置 | 功能场景描述（摘要） |
|-----|------|--------|----------|----------|----------------------|
| 654 | 收据 | 全局设置 | 收据 | 下单后自动打印纸质订单收据 | 设置下单后是否自动打印纸质订单收据 |
| 500 | 收据 | 服务设置 | 收据 | 部分付款自动打印纸质订单收据 | 设置部分支付的订单是否自动打印纸质订单收据 |

| 261 | 收据 | 收据打印 | 打印设置 | 付款前打印收据 | 设置订单支付前是否自动打印收据小票 |
