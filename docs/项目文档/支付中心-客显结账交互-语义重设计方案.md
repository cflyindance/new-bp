## 3. 目标结构

### 3.1 支付中心组序（9 组 → **10 组**）

```text
… → card-fees（卡支付规则与合规）
  → checkout-tip-card-order（小费与刷卡顺序 · 全终端 SSOT）
  → cds-checkout-ux → guest-self-checkout-ux → paypad-checkout
```

**为何单独成组**：同一能力（先小费还是先刷卡）在 **CDS / Kiosk / PayPad** 上**允许不同策略**，不应挂在「结账与交互」单组内，也不应在三组各留一条平行项。

### 3.2 `card-fees` 扩展（吸收 243）

| groupKey | groupTitle（新） | 条数 | seq |
|----------|------------------|------|-----|
| `card-fees` | **卡支付规则与合规** | 3 | 242, **243**, 454 |

组内顺序：**242 产线最低消费 → 243 产线签名门槛 → 454 卡付加价策略**

组首说明补充：

> 242/243 为**全产线**卡支付规则；客显/扫码/Paypad **签名页开关**见下方三组结账交互（464 / 497 / 664）。

### 3.3 `cds-checkout-ux` 收敛

| groupKey | groupTitle | 条数 | seq |
|----------|------------|------|-----|
| `cds-checkout-ux` | 结账与交互 | 4 | 463, 464, 465, 669 |

组内顺序：**463 小费 → 464 签名 → 465 小票 → 669 刷卡签购单**

组首说明：

> 本组仅 **客显屏（CDS）** 结账步骤展示开关。签名金额门槛（243）见「卡支付规则与合规」；**小费与刷卡先后顺序**见上方 **「小费与刷卡顺序」** 组（seq 9）。

### 3.4 `checkout-tip-card-order`（新增）

| groupKey | groupTitle | 条数 | seq |
|----------|------------|------|-----|
| `checkout-tip-card-order` | **小费与刷卡顺序** | 1 | **9** |

组首说明：

> 控制 **CDS、Kiosk、PayPad** 在卡付结账时，**先进入小费页还是先刷卡**。各产线可独立配置；POS / eMenu 不涉及食客自助刷卡流程，不在此矩阵。

**UI 草图（产线 × 二选一）**：

```text
┌─ 小费与刷卡顺序（9）──────────────────────────────────────┐
│  各终端可分别设置；原 seq 495（扫码）、663（Paypad）已合并于此。 │
├──────────────┬──────────────────────────────────────────────┤
│ 客显（CDS）   │  ○ 刷卡前选择小费   ● 刷卡后选择小费          │
│ Kiosk        │  ● 刷卡前选择小费   ○ 刷卡后选择小费          │
│ PayPad       │  ○ 刷卡前选择小费   ● 刷卡后选择小费          │
└──────────────┴──────────────────────────────────────────────┘
```

**Storage**：`9-tip-card-order-by-line` → `{ cds, kiosk, paypad }` 各为 `tip-before-card` | `tip-after-card`。

## 4. 分项方案

### 4.1 243 — 迁回 `card-fees`

| 项 | 内容 |
|----|------|
| **理由** | 按产线配置，非 CDS 专属；与 242 同属「卡交易规则」 |
| **UI** | 已有产线表格（`243-card-signature-min-by-line`），随组迁移，渲染位置改到 card-fees 组 **242 之后** |
| **文案** | 标题保持「信用卡签名最低金额」；场景强调「各终端签名页开启且达到该金额才强制签名」 |

### 4.2 8 + 464 — 合并为 SSOT

| 项 | 内容 |
|----|------|
| **保留 seq** | **464** |
| **新标题** | 客显签名页（或保留「签名」） |
| **场景描述** | 客显屏是否在卡付后展示电子签名页；低于 243 产线门槛时可跳过（若 243>0）。原 seq 8「刷卡签名」已合并。 |
| **排除** | **8** — catalog 排除，legacy 开关迁移至 464 storage |
| **UI** | 标准 toggle；组内可选一行灰色说明引用 243 |

**迁移逻辑**（localStorage）：

```text
若 464 无值且 8 有值 → 464 := 8
```

### 4.3 463 — 小费页（政策 vs 展示）

| 项 | 内容 |
|----|------|
| **保留** | 客显组 |
| **职责** | ① 是否展示小费页 ② 客显专属展示项（如 No-Tip 是否展示，若终版有则后续补） |
| **禁止重复** | **不在 463 编辑预设比例/金额** — SSOT 为 `tip-policy` **237**（及 295/296 收据类） |
| **UI 草图** | |

```text
┌─ 客显小费页（463）────────────────────────────┐
│  [开关] 在客显展示小费选择页                      │
│  预设选项来源：○ 同步结账页预设（237）  ○ 独立  │
│  （默认：同步 237；选「独立」时才展开客显专用预设表）│
│  小费计算规则见「小费政策与计算」                  │
└────────────────────────────────────────────────┘
```

首版可简化为：**开关 + 一句说明「预设与 237 同步」**，不做独立预设表，避免与 237 双 SSOT。

### 4.4 9 + 495 + 663 — 小费与刷卡顺序（产线矩阵 SSOT）

| 项 | 内容 |
|----|------|
| **保留 seq** | **9**（catalog 唯一入口） |
| **排除** | **495**（扫码端）、**663**（Paypad）— 语义已并入 9 |
| **新标题** | **小费与刷卡顺序**（去掉「签字」，避免与 464 签名页混淆） |
| **场景描述** | 控制 CDS、Kiosk、PayPad 在卡付结账时，小费选择页与刷卡步骤的先后顺序；**各产线可不同** |
| **选项（每产线）** | `tip-before-card` 刷卡**前**选择小费 · `tip-after-card` 刷卡**后**选择小费 |
| **产线列** | `cds` 客显 · `kiosk` Kiosk · `paypad` PayPad（**不含** pos / emenu） |
| **UI 模式** | 复用 242/243 **按产线表格**布局；每行两个互斥 radio（非全局单选） |
| **Runtime 契约** | CDS 读 `byLine.cds`；Kiosk 读 `byLine.kiosk`；PayPad 读 `byLine.paypad` |

**Legacy 迁移**（localStorage）：

```text
若 9-tip-card-order-by-line 无值：
  cds   ← seq 9 旧全局 choice（tip-before-card / tip-after-card）
  kiosk ← seq 495 旧 choice
  paypad← seq 663 旧 choice
若某产线仍无值 → 默认 tip-before-card
```

**与 242/243 的差异**：242/243 覆盖 POS/Kiosk/eMenu/PayPad **四产线金额**；本项仅 **三终端结账 UX 产线**，因 POS 店员操作、eMenu 非典型自助刷卡，不纳入矩阵。

### 4.5 465 — 小票

| 项 | 内容 |
|----|------|
| **保留** | 客显组（展示为三产线矩阵） |
| **边界** | 结账后小票/收据**输出方式**（非模板/样式；模板在打印中心） |
| **通道** | 每产线三行：**纸质小票**、**电子小票（短信）**、**电子小票（邮箱）**；各通道手动 / 自动 / 不输出 |
| **互斥** | 同一产线仅一个通道可为「自动」 |
| **合并** | 原 **501** → Kiosk 短信通道；**665** → PayPad 邮箱通道；历史 `electronic` 单字段按产线迁移（CDS/Kiosk→短信，PayPad→邮箱） |

### 4.6 669 — 刷卡签购单（纸质）

| 项 | 内容 |
|----|------|
| **迁入** | 自硬件 **669/670**（原 PayPad 终端项） |
| **通道** | 每产线两行：**Merchant Copy（商户联）**、**Customer Copy（客户联）**；各联次手动打印 / 自动打印 / 不打印 |
| **互斥** | 商户联与客户联**独立**配置（可同时为自动） |
| **合并** | **670** 排除 catalog，并入 Merchant Copy 行；PayPad 历史开关迁移至 PayPad 产线列 |
