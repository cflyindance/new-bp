## 2. 现状语义诊断

### 2.1 三条配置的真实语义

| seq | 表面归类 | 实际语义 | 配置类型 |
|-----|----------|----------|----------|
| **554** | 会员 · 登录引导图 | 登录页是否展示引导**广告** + **换图** | 产线 UI 显隐 + 素材 |
| **615** | 会员 · 展示设置 | 登录流程是否弹出**付费会员 upsell 弹窗** | 营销 CTA（依赖付费会员产品） |
| **650** | 会员 · 权益会员 | 是否展示**权益会员说明海报** | 产线 UI 显隐 + 海报内容 |

共同点：**食客端登录/点餐路径上的展示与营销**，不是会员账户、积分规则或登录校验策略。

### 2.2 与会员设置其他组的分界

| 维度 | `login-verification` | `member-guidance-benefits`（现状） |
|------|------------------------|-------------------------------------|
| 商户在配置 | 要不要登录、怎么校验 | 登录页/路径上放什么广告 |
| 抽象层 | **流程策略（gate）** | **触点营销（chrome）** |
| 类比 | 504「是否展示手机号页」 | 554「登录页是否展示引导广告」 |

504/506 已迁前厅 `guest-order-rules`；648/649 海报已迁 `/marketing/ads`。**554/615/650 按同一原则不应留在会员设置滑层。**

### 2.3 组名问题

「会员引导与**权益展示**」把两件事捆在一起：

- **554 + 615** → 登录漏斗拉新 / upsell（营销）
- **650** → 权益 tier 说明（产品教育 + 海报）

商户在一个组里同时改「广告弹窗」和「权益说明」，心智混乱；且每项都隐含**内容编辑**（图/海报），设置滑层只能做开关，功能不完整。

### 2.4 与现有功能页重复

| seq | 设置滑层项 | 已有功能页（navigation） |
|-----|------------|--------------------------|
| 554 | 登录引导图设置 | `/ordering/kiosk/login-guide` |
| 615 | 展示付费会员登录引导 | `/members/card/paid-config` |
| 650 | 权益会员介绍 | `/ordering/tablet/membership-benefits` |

三条在 B 端已有更合适的**产线/业务配置入口**；设置 catalog 重复占位。

## 3. 目标结构

### 3.1 会员中心设置：三组收敛

| groupKey | groupTitle | 条数（v1.4 后） | 职责 |
|----------|------------|-----------------|------|
| `member-account-system` | 会员账户与卡体系 | 4 | 云/本地、使用模式、有效期、查卡 |
| `points-rewards` | 积分规则与兑换商品 | 4 | 积分展示与积分菜兑换策略 |
| `login-verification` | 登录注册与信息校验 | 5 | 身份策略、验证码、注册字段、隐私默认 |
| ~~`member-guidance-benefits`~~ | ~~会员引导与权益展示~~ | **0（解散）** | — |

### 3.2 迁出后 SSOT 地图

```text
554  登录引导图     →  Kiosk · 登录引导图（开关 + 换图 + 产线说明）
615  付费会员弹窗   →  会员 · 付费会员配置（套餐 SSOT + 「登录时展示引导」子开关）
650  权益会员海报   →  eMenu · 权益会员（展示开关 + 海报/文案维护）
```

引导图**图片文件**若需统一素材库，与封面/屏保一样走**素材中心**；产线页只引用素材 ID，不在设置滑层重复「换图」。

## 4. 分项方案

### 4.1 554 — 登录引导图设置

| 项 | 内容 |
|----|------|
| **catalog 动作** | **排除**（`settings-catalog-exclusions.mjs`） |
| **SSOT 宿主** | `/ordering/kiosk/login-guide` |
| **排除理由** | 登录页广告位 + 换图；属 Kiosk 终端壳层配置，非会员 hub 策略 |
| **素材** | 引导图文件 → 素材中心（与封面图、屏保同族）；Kiosk 页选择/上传 |
| **产线** | 当前 navigation 仅 Kiosk 有独立入口；eMenu 若共用登录引导，宜在同页增加产线 Tab 或 eMenu 对称页（后续产品项，不阻塞 v1.4 catalog 瘦身） |

**功能页建议字段（paid-config 对称参考）：**

- 开关：是否展示登录引导
- 图片：选择/上传引导图（或从素材库选）
- 说明：与 `login-verification` 623「点单前身份策略」的优先级（先引导图 vs 先登录页）

---

### 4.2 615 — 展示付费会员登录引导

| 项 | 内容 |
|----|------|
| **catalog 动作** | **排除** |
| **SSOT 宿主** | `/members/card/paid-config` |
| **排除理由** | 付费会员 **upsell 弹窗**；强依赖「付费会员套餐」产品，应作为套餐配置的子选项，非全局设置 |
| **原模块语境** | 终版落在「展示设置」，与「展示品牌 LOGO」「展示菜品编号」同族 → 本质是**食客端展示**，但因绑定付费会员产品，归 **paid-config** 优于前厅 generic 展示组 |

**功能页建议字段：**

- 区块：「登录触达」
- 开关：会员登录/注册时展示付费会员引导弹窗
- 依赖：仅当已配置至少一个付费会员套餐时可开启；否则灰显 + 链至套餐编辑
- 与 **554** 互斥/优先级说明（避免登录连弹两个广告）

---

### 4.3 650 — 权益会员介绍

| 项 | 内容 |
|----|------|
| **catalog 动作** | **排除** |
| **SSOT 宿主** | `/ordering/tablet/membership-benefits` |
| **排除理由** | 「海报」类展示 + 权益 tier 说明；与 648/649 下单页海报同族，宜在产线页完整维护 |
| **典型行为** | 非权益会员登录后展示权益说明海报（以产品定义为准） |

**功能页建议字段：**

- 开关：是否展示权益会员介绍
- 海报/文案：图片 + 标题（或富文本摘要）
- 产线：eMenu 宿主；Kiosk 若需同能力，产线 Tab 扩展

**边界**：若「权益会员」仅为 POS 本地 tier、云上无对应能力，则整页标记为遗留/只读，catalog 仍排除，不在设置滑层保留孤儿开关。

- **不修改** `配置归类-终版.md` 原文；hub catalog 与功能页 SSOT 分离。
